# 教室配当仕様案

本書は、教室配当ロジックを「一律のポイント制」から、「必須条件・準必須条件・希望条件を分けて扱う配当方式」へ整理するための仕様案である。

対象は自動配当であり、手動のドラッグ&ドロップによる例外配置は別扱いとする。

## 1. 基本方針

自動配当では、以下を原則とする。

- 同一時限・同一教室への重複配当はしない
- 定員不足の教室には配当しない
- 必須機材が欠ける教室には配当しない
- 連続講時の授業は、同じ教室でまとめて扱う
- 自動配当中に、暗黙の教室分割や定員超過配当はしない
- 条件を満たす候補がなければ、無理に押し込まず未配当にする

つまり、自動配当は「安全に成立する配置を作る」ことを最優先とし、成立しないものは未配当に残して理由を明示する。

## 2. ルールの階層

ルールは以下の3層に分けて扱う。

### 2-1. 絶対必須

これらは1つでも満たさなければ、その教室候補を即時除外する。

- 時間重複なし
- 連続講時同室
- 定員不足なし
- 必須機材一致
- 配当対象外教室は除外

備考:
- 「連続講時同室」は加点対象ではなく、配当単位そのものとして扱う
- 「入りきらない」は例外配当せず、必ず未配当とする

### 2-2. 準必須

原則として守るが、設定上許可されている場合のみ、最後の段階で例外配当を認める。

- 教室タイプ一致
- 春秋同一教室

推奨設定:
- 教室タイプ不一致配当: ON/OFFを選択可能にする
- 春秋同一教室の例外配当: 将来的にON/OFF化可能だが、まずは「最終段階でのみ緩和」に固定でもよい

例外緩和の優先順:
1. 春秋同一教室を崩す
2. 教室タイプ一致を崩す

理由:
- 教室タイプ不一致は授業実施そのものに影響しやすいため、春秋同一教室より重く扱う

### 2-3. 希望条件

配当可否は左右しないが、候補比較の順序として使う。

優先順は以下を推奨する。

1. 同一教員連続授業
2. 希望機材の充足度
3. 適切な教室サイズ
4. 建物希望
5. 過年度教室優先

補足:
- 同一教員連続授業は、建物希望より強く、サイズ適合よりも強く扱う
- 教室サイズは「入りきること」が必須であり、「ちょうどよい大きさ」は希望条件である

## 3. 配当単位

### 3-1. 連続講時

連続講時の授業は、開始講時から終了講時までを1つの配当単位として扱う。

- 候補教室は、関係する全講時で空いていなければならない
- 途中の講時だけ別教室にする自動配当は行わない
- 1講時目だけを先に配当することはしない

### 3-2. 複数教室必要科目

同一授業コード・同一曜日時限で複数教室が必要な科目は、1科目として管理しつつ、必要教室数ぶんの枠を持つ。

- 必要数を満たさない限り「部分成立」ではなく未配当扱いとする
- ただしUI上は「1/2室配当」など途中経過を表示してよい
- 自動配当の最終結果としては、必要数を満たしたものだけを配当済みとするのが望ましい

備考:
- 実装負荷を抑えるため、初期段階では「不足分が1室でもあれば未配当リストに残す」方式でよい

### 3-3. 春秋ペア

春秋同一教室を狙う対象は、可能であれば `linkedSubjectId` など明示的なペア情報で結び付ける。

- 教員名・曜日・講時だけでの推定は補助的扱いとする
- 明示ペアがある場合は、それを優先する

## 4. 科目の処理順

自動配当では、単純な「優先度順」ではなく、以下の順で並べる。

1. 科目優先度
2. 配当困難度
3. 前回までの未配当履歴
4. 入力順またはCSV取込順

### 4-1. 配当困難度

配当困難度は、次の要素で高くする。

- 厳密条件を満たす候補教室数が少ない
- 必須機材が多い
- PC室やゼミ室など、希少な教室タイプを必要とする
- 必要定員が大きい
- 連続講時である
- 春秋同一教室を持つ
- 必要教室数が複数である

原則:
- 同じ優先度なら「入りにくい科目」を先に配当する

### 4-2. 未配当履歴

未配当履歴は、同率時の救済要素として使う。

- 前回・前々回も未配当だった科目は、少しだけ先に処理する
- ただし未配当順の固定FIFOにはしない

理由:
- FIFOだけにすると、難しい科目が永続的に不利になる可能性がある
- 毎回「優先度 + 困難度」を再計算し、その上で未配当履歴を補助的に使う

## 5. 候補教室の比較方法

候補比較は、総合ポイントの足し算ではなく、上位条件から順に比較する。

推奨する比較順は以下の通り。

1. 例外なしで成立しているか
2. 同一教員連続授業を満たすか
3. 希望機材の充足度が高いか
4. 教室サイズが適正範囲に近いか
5. 建物希望を満たすか
6. 過年度教室に一致するか
7. 余剰定員が少ないか

つまり、

- 建物希望が合うからといって、同一教員連続授業より上にはならない
- 過年度教室だからといって、希望機材の不足を打ち消さない

という比較順にする。

## 6. 配当フロー

配当は次の4段階で行う。

### 6-1. 第1段階: 厳密配当

絶対必須と準必須をすべて満たす教室だけを候補にする。

- 候補があれば、その中で希望条件を比較して最良候補を選ぶ
- この段階では例外配当はしない

### 6-2. 第2段階: 例外付き配当

第1段階で候補が0件のときだけ、準必須の緩和を順番に試す。

推奨順:
1. 春秋同一教室を緩和
2. 教室タイプ一致を緩和

ルール:
- 教室タイプ不一致配当がOFFなら、2は試さない
- 緩和候補が見つかった場合は、「例外配当」であることを結果に残す

### 6-3. 第3段階: 局所再調整

第2段階でも入らない場合、1回または少数回の玉突き再調整を試す。

対象:
- ある教室に現在入っている科目を別候補へ移せば、全体として配当可能になる場合

制限:
- 深い探索は行わず、1段または2段の局所入替までに留める
- 絶対必須を壊す入替はしない
- 例外レベルが悪化する入替はしない

成立条件:
- 未配当件数が減る
- または、より高優先度の科目を救済できる

### 6-4. 第4段階: 未配当

第3段階まで試しても成立しない場合は、未配当に送る。

- 重複配当はしない
- 定員超過配当はしない
- 必須機材不足配当はしない

## 7. パターン別の扱い

### 7-1. 同率科目が同じ教室を取り合う場合

同率科目の優先順は以下とする。

1. 厳密候補数が少ない科目
2. 必須機材・特殊タイプ条件が重い科目
3. 定員が大きい科目
4. 連続講時や春秋条件を持つ科目
5. 未配当履歴が多い科目
6. 取込順

### 7-2. 入る教室はあるが、適切サイズではない場合

- 定員を満たしていれば配当対象に含める
- 適正範囲内があればそちらを優先する
- 適正範囲外しかなければ、その中で余剰が最も少ない教室を選ぶ

### 7-3. 教室タイプが合う教室がない場合

- 例外配当がOFFなら未配当
- 例外配当がONなら、最終段階でのみ不一致配当を許可する
- 不一致配当時は「タイプ不一致」の理由を残す

### 7-4. 春秋同一教室が取れない場合

- 原則は同一教室を優先する
- 同一教室候補が0件で、かつ例外配当段階に入った場合のみ別教室を許可する
- このとき「春秋分離」の理由を残す

### 7-5. 必須機材は満たすが希望機材が一部不足する場合

- 配当は可能
- 希望機材の充足率が高い教室を優先する
- 充足率が低い場合は警告または理由表示の対象にする

### 7-6. 必要教室数を満たせない場合

- 不足分が出た時点で未配当扱いにする
- 1室だけ先に確保して「部分配当済み」とはしない
- ただし内部的には確保候補を保持して、再調整時の材料にしてもよい

### 7-7. 候補はあるが、すべて低優先度科目で埋まっている場合

- 局所再調整を試す
- 高優先度科目を救済でき、かつ押し出される側にも代替候補がある場合は入替を許可する
- 代替候補がなければ現状維持とし、元の科目を未配当に送る

## 8. 未配当の分類

未配当は理由を分けて表示する。

推奨分類:

- `U1`: 絶対必須条件を満たす教室が存在しない
- `U2`: 教室タイプ一致を満たす教室が存在しない
- `U3`: 春秋同一教室を維持できない
- `U4`: 必要教室数を満たせない
- `U5`: 局所再調整でも成立しない

表示順:

1. 高優先度
2. 絶対必須違反に近いもの
3. 候補数が少ないもの
4. 未配当履歴が多いもの
5. 取込順

## 9. 設定画面で持たせる項目

最低限、以下の設定があると運用しやすい。

- 教室タイプ不一致配当を許可する
- 希望機材の重みを調整する
- 適切サイズの理想範囲を調整する
- 同一教員連続授業の優先度を調整する
- 建物希望・過年度教室の優先度を調整する

固定でよい項目:

- 重複配当しない
- 定員不足配当しない
- 必須機材不足配当しない
- 連続講時を分割しない

## 10. 推奨する初期運用

初期設定としては、次を推奨する。

- 教室タイプ不一致配当: OFF
- 春秋同一教室: 原則維持、ただし最終段階でのみ緩和
- 同一教員連続授業: 強い希望
- 適切サイズ: 希望条件
- 建物希望: 弱い希望
- 過年度教室優先: 弱い希望

この設定なら、

- 授業実施上まずい配置は自動で避ける
- どうしても足りない場合だけ例外配当に落とす
- 例外でも無理なものは未配当に残して人が判断できる

という運用がしやすい。

## 11. 実装上の注意

現行実装では、いくつかの条件が「必須に見えて実際は加点」になっている箇所がありうるため、次の順で整理するのが望ましい。

1. 連続講時を加点ではなく配当単位に変更する
2. 準必須条件の緩和順を明示する
3. 希望条件を単純加点ではなく比較順で扱う
4. 未配当理由を構造化して保持する
5. 必要なら局所再調整を追加する

以上を満たすことで、「理論上は点数が高いが、実務上は困る配置」が起きにくくなる。
