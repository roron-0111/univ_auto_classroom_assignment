# クラウド同期・書き込みロック設計

## 1. 目的

現在のクラウド同期は、ローカルとクラウドを全量スナップショットで扱っている。
このため、同時編集や古い画面からの共有で、意図しない上書きが起きやすい。

そこで、操作を次の2つに分ける。

- `ローカル → クラウドへ書き込み`
- `クラウド → ローカルに取得`

旧 `更新(共有)` は廃止し、方向を明示する。

## 2. 基本方針

- 書き込みと取得を分離する
- ロックは **書き込み時のみ** 使う
- 取得はロック対象外とする
- 既存のローカル作業を不用意に消さないよう、取得前には上書き確認を入れる
- 共有前にはクラウド最新版とローカルの差分を確認し、必要なら CSV 退避を促す

## 3. ボタンの役割

### 3-1. `ローカル → クラウドへ書き込み`

ローカルで編集した現在状態をクラウドへ保存する。

この操作の前に以下を行う。

1. クラウドの最新版を取得する
2. ローカルの最終取得時点と差分を確認する
3. 差分がある場合は警告モーダルを表示する
4. 必要に応じて CSV エクスポートを促す
5. 続行時のみ短期ロックを取得する
6. 保存成功後にロックを解除する

### 3-2. `クラウド → ローカルに取得`

クラウド側の最新データをローカルに反映する。

この操作はロック対象外とする。
ただし、ローカルに未保存の変更がある場合は、上書き前に確認を出す。

## 4. ロック設計

ロックは **書き込み中だけの短期ロック** とする。

### 4-1. ロックの目的

- 同時書き込みの衝突を避ける
- 共有保存中に別ユーザーが入り込むことを抑える
- 長時間の閲覧ロックは避ける

### 4-2. ロックが必要な場面

- `ローカル → クラウドへ書き込み` を実行する直前

### 4-3. ロックが不要な場面

- `クラウド → ローカルに取得`
- 一覧閲覧
- 編集画面の参照

## 5. ロック解除のタイミング

ロックは長時間保持しない。解除タイミングは次の通り。

### 5-1. 正常終了時

- クラウドへの書き込みが成功した直後に解除する

### 5-2. 異常終了時

- 書き込みが失敗した場合も `finally` 相当の処理で解除する
- ネットワークエラーや例外で保存が中断した場合も解除する

### 5-3. 取り消し時

- 差分警告モーダルでキャンセルした場合は、ロックを取得しない
- したがって解除も不要

### 5-4. 保険としての失効

- 端末終了や予期しない切断に備え、ロックには短い有効期限を持たせてもよい
- ただし本設計では、あくまで **書き込み中の短期ロック** を主軸とする

## 6. 差分警告

`ローカル → クラウドへ書き込み` を実行する前に、クラウド側との差分を示す。

### 表示内容

- `更新データがあります`
- `更新前にローカルデータを CSV エクスポートしてください`
- 変更件数の要約
  - 授業
  - 教室
  - 配当
  - 設定

### 差分粒度

最初は件数ベースで十分とする。

- 追加
- 変更
- 削除

必要なら後から科目名や教室名を含めた詳細差分を追加する。

## 7. データメタ情報

クラウド側には、将来の競合検出や差分確認のため、最低限次の情報を持たせる。

- `revision`
- `updatedAt`
- `updatedBy`
- `campus`

`revision` は保存のたびに増える単調増加値が望ましい。

## 8. 取得時の扱い

`クラウド → ローカルに取得` は書き込みではないため、ロックを使わない。

ただし、ローカルに未保存の変更がある場合は、上書き前に確認する。
この確認は、ローカル作業を誤って失わないための安全策である。

## 9. 触る主なファイル

- `src/App.tsx`
  - ボタン分離
  - 書き込み前差分モーダル
  - 取得時の上書き確認
  - ロック中の書き込み抑止
- `src/utils/useCloudSync.ts`
  - クラウド読み書き
  - `revision` / `updatedAt` の管理
  - 短期ロックの取得・解除
- `src/components/`
  - 差分警告モーダル
  - 必要なら取得確認モーダル
- `src/types_cloud.ts`
  - クラウドメタ情報の型追加

## 10. 実装順

1. ボタンを `ローカル → クラウドへ書き込み` と `クラウド → ローカルに取得` に分ける
2. 旧 `更新(共有)` を削除する
3. 書き込み前の差分警告を入れる
4. 書き込み時だけ短期ロックを入れる
5. 取得時の上書き確認を入れる
6. `revision` の確認を追加する
7. 必要なら差分の詳細表示を増やす

## 11. 非目標

- 長時間の編集ロック常駐
- ブラウザ終了で必ず解除される前提
- 項目単位のマージ同期
- リアルタイム双方向同期

この設計では、まず安全性と分かりやすさを優先し、実装を段階的に進める。
