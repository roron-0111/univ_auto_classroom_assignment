# ガイド設計

## 目的

操作を一度に説明せず、利用者が選んだ作業だけを短い言葉で案内する。
必要な画面はガイドが自動で開く。

## 項目

- 画面の見方: 未配当、時間割表、曜日、絞り込み、凡例
- 授業を準備する: 授業管理、CSVインポート、授業一覧、学部・管轄、CSVエクスポート
- 教室を準備する: 教室管理、CSVインポート、教室一覧、配当対象外、CSVエクスポート
- 自動で配当する: 配当ルール設定、対象、配当モード、希望条件、機材条件、実行ボタン
- 手で調整する: 未配当、絞り込み、時間割表、凡例
- 保存・取得する: 書込、取得、ログアウト
- 困ったとき: 凡例、配当クリア、再配当前の確認

## 画面切り替え

各ステップは `view` を持つ。

- `main`: 通常画面
- `subjects`: 授業管理
- `classrooms`: 教室管理
- `rules`: 配当ルール設定

ガイドはステップ移動時に該当画面を開き、対象要素を再探索してハイライトする。
項目選択へ戻る時と終了時は通常画面へ戻す。

## 注意レベル

- `info`: 通常確認
- `careful`: CSV読込、保存、取得、再配当など、実行前確認が必要な操作
- `danger`: 自動配当実行、配当クリアなど、結果が大きく変わる操作

`careful` と `danger` は、ガイド中に自動クリックしない。

## 文言

- 見出しは短くする
- 本文は2文以内にする
- 業務用語は残しつつ、意味が伝わる表現にする
- キャラクターは置かず、控えめなガイドアイコンだけを使う
