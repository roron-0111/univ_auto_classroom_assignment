## 現在地メモ

- `cloudDiff.ts` を新規化し、差分比較と差分CSV生成の土台を分離済み
- `書込` は、既存クラウドと比較して変更のない collection をスキップし、差分を反映する段階まで進行
- `取得` は、差分確認モーダルを挟んでクラウド内容をローカルへ反映する流れに整理済み
- 差分CSVは、取得側の確認モーダルから `対象` 列つきで出力する流れを維持
- `書込` の競合検出は revision 比較と閲覧のみ案内まで完了し、古いローカル状態の上書きを止める
- `取得` 側の差分詳細と CSV 導線は整理済み
- 差分CSV は `CloudReadWarningModal` から出力し、`キー` は出さない
- root の `npm run lint` / `npm run build` は通過済み
- Phase 8 は静的確認済み。ブラウザでの往復確認だけが残っている

## メモ

- `cloudDiff.ts` に差分比較と CSV 出力の土台を切り出した
- `取得` と `書込` は、どちらも全量置換ではなく差分反映前提で作る
- 現状の書込は、既存クラウドと比較して変更のない collection をスキップし、変更のある collection のみを更新する段階まで進んだ
- 差分がまったく無い場合は、書込完了の代わりに「変更はありませんでした。」を返す
- 競合検出は、差分反映より後ではなく、**実装の早い段階で組み込む**
- 差分CSV は `種別 / 操作 / 対象 / ローカル / クラウド` の列で出力する
