# 総合チェックリスト

本チェックリストは、教室配当・調整アプリの完成後レビューを漏れなく進めるための監査項目です。
最優先観点は、配当ロジックの整合性、既存データ互換性、同期処理の安全性、実務上の使いやすさです。

## 進め方

### 推奨フェーズ

1. **棚卸し**: 仕様、型、保存キー、同期データ、CSV列、主要画面、既存テストを一覧化する。
2. **静的確認**: 型、lint、build、Firestore rules、依存関係、秘匿値の混入を確認する。
3. **互換性確認**: 旧localStorage、旧CSV、旧Cloud単一ドキュメント、既存配当データを壊さないか確認する。
4. **配当ロジック確認**: 自動配当、未配当理由、複数教室、連続講時、春秋ペア、玉突き再調整を重点確認する。
5. **同期安全性確認**: 複数端末、古いローカル、書き込みロック、差分CSV、キャンパス分離を重点確認する。
6. **画面・実務確認**: 授業管理、教室管理、時間割、未配当、設定、モーダル、CSV操作を実務フローで確認する。
7. **セキュリティ確認**: 認証、Firestore権限、固定アカウント方式、ブラウザ保存、ログ、CSV取り扱いを確認する。
8. **性能・耐久確認**: 大量データ、低速回線、連打、タブ並行、リロード、途中失敗を確認する。
9. **回帰確認**: 既存E2Eと手動シナリオを通し、未解決項目を分類する。
10. **最終判定**: リリース可、要修正、運用注意、保留のいずれかに分類して記録する。

### 実施単位

- 1回目: 静的・互換・セキュリティの机上チェック。
- 2回目: 配当ロジックだけを重点的に確認。
- 3回目: Cloud同期とキャンパス分離だけを重点的に確認。
- 4回目: UI操作、CSV、実務フローを実データに近い形で確認。
- 5回目: 見つかった問題の修正後、影響範囲だけを再確認。

### 記録形式

各項目は次の形式で記録する。

- 状態: `未着手` / `OK` / `NG` / `N/A` / `保留`
- 証跡: コマンド、スクリーンショット、CSV、Cloud状態、対象データ名
- メモ: 再現条件、期待値、実際の結果、対応方針

### チェック反映ルール

- 自動確認できた項目は `[x]` にする。
- 手動確認待ちの項目は `[ ]` のまま残す。
- 不具合は修正せず `要修正` として監査結果に記録する。
- チェックリスト外の確認や軽微な派生対応は、監査結果メモに「抱き合わせ候補」として残すだけにする。
