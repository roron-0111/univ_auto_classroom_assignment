# CSV 監査チェックリスト

CSV の入出力と差分出力について、列名・値・往復性を確認するためのチェックリストです。  
修正済みのものは `[x]`、まだ確認が必要なものは `[ ]` として管理します。

## 1. 差分 CSV（クラウド取得時）
- [x] `書込` ではなく、`取得` 時の差分確認でだけ CSV を出す
- [x] A列/B列は `種別` / `操作` として差分の種類を示す
- [x] C列はローカル側の状態を示す
- [x] D列はクラウド側の状態を示す
- [x] C列・D列で `なし→あり` / `あり→なし` を可視化する
- [x] E列は教室名を表示する
- [x] F列は曜日を表示する
- [x] G列は講時を表示する
- [x] H列は `コード_科目名` を表示する

## 2. 授業 CSV インポート
- [x] 必要教室数が 2 以上でも 1 に潰れない
- [x] `希望建物` と `棟希望` を受け入れられる
- [x] `過去教室` / `教室(過去教室)` / `過去教室(区切り)` を受け入れられる
- [x] 過去教室は `,` 区切りでも復元できる
- [x] 配当教室は `教室ID` を正として受ける
- [x] `教室ID` が現在の教室マスタに無い行は弾く
- [x] `◎` / `○` の受け入れルールが崩れていない

## 3. 授業 CSV エクスポート
- [x] `必要教室数` が export で保持される
- [x] O列ヘッダーが `希望建物` になっている
- [x] 過去教室の列名が `過去教室` で出る
- [x] 配当教室 ID 列が教室名ではなく `教室ID` で出る
- [x] 教室ID の旧データが残っていても、可能な範囲で actual ID に正規化して出す

## 4. 教室 CSV インポート
- [x] `○` と `◯` を受け入れられる
- [x] 可動・設備タグの意味が崩れない
- [x] `教室ID` を ID として扱う
- [x] 現在のキャンパス以外のレコードは弾く

## 5. 教室 CSV エクスポート
- [x] `教室ID` は教室名ではなく実 ID で出る
- [x] ヘッダーがインポート側と整合している
- [x] 設備タグの round-trip が壊れない

## 6. 実機確認メモ
- [x] `書込 → 取得 → 再書込` の往復確認
- [x] 追加のみの差分が反映されることの確認
- [x] 更新のみの差分が反映されることの確認
- [x] 削除のみの差分が反映されることの確認
- [x] 同一ユーザーの別端末で古いデータが上書きされないことの確認

## 7. 補足
- 差分 CSV は allocations の差分を人間が追いやすい形に整える。
- 授業 CSV の import/export は、旧ファイルとの互換性を壊さない範囲で列名の別名を受ける。
- 教室 CSV は、`○` / `◯` のような既存データの表記も吸収する。
